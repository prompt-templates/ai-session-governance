# v3.0 Release Notes

Date: 2026-04-25 (UTC)

## Highlights

- AGENTS.md cut from 734 to 487 lines (−247, −33.7%) while preserving every rule and grep-anchor.
- INIT.md FILE 1 mirror reduced from 951 to 704 lines.
- System-prompt token cost per session boot reduced ~15.6% (from ~11,715 to ~9,890).
- Legacy quarantine mechanism: 89 historical drift-defense checks (R11/R26/R27 series) isolated into `docs/qa/legacy_checks.sh`, auto-chained from the main harness.

- New regression checks (`R29` series) enforce that READMEs, `docs/site/index.html` stat counter, and `docs/qa/LATEST.md` reflect the latest stable tag.

## Upgrade path

Existing v2.x users:

- Re-run `INIT.md`. The `§5a` `INSTALL_ROOT_OK` + `INSTALL_WRITE_OK` two-stage gate auto-backs up your current files to `dev/init_backup/<timestamp>/` (now including `SESSION_STATE_DETAIL.md` and `PROJECT_MASTER_SPEC.md` if you have them).
- Or merge the v3.0 diff manually into your existing AGENTS.md if you have local customizations.

## Industry standards alignment

Industry consensus (HumanLayer, aihero.dev, Anthropic best practices) recommends ≤200 lines for AGENTS.md / CLAUDE.md to maintain ~89% instruction adherence rate (vs ~35% for verbose). v3.0 brings AGENTS.md from severely overweight (734 lines) to a compact 487 — the residual is necessary rules + behavior-dependent ASCII Visual Cue blocks. Reaching ≤200 requires a `plays/` extraction architecture (separate v3.x roadmap, out of scope here).
